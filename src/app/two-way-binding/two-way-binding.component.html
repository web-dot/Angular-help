Data binding

Angular supports two-way data binding, a mechanism for coordinating the parts of a template with the parts of a component.
Add binding markup to the template HTML to tell Angular how to connect both sides.
Following shows four forms of data binding, each form has a direction: to the DOM, from the DOM or both

interpolation braces ::::: DOM <---------- COMPONENT
[property] = "value" ::::: DOM <---------- COMPONENT

(event) = "handler" :::: DOM --------------> COMPONENT

[(ngModel)] = "property" :::: DOM <----------------> COMPONENT

This example uses three of tese forms

  • {{name}} (interpolation: dom <-------- component)



    • The name interpolation displays the component's name property value within the element.
    • The [disabled] property binding passes the value of isUnchanged to the disabled property of the button.
    • The click event binding calls the components handleEvent() method when user clicks the button.
      • Event binding

    Binding to events
    To bind to an event you use the Angular event binding syntax.
    This syntax consists of a target event name within parentheses to the left of an equal sign, and a quoted template statement to the right.

    <button (click)="onSave()">Save</button>

    Two-way data binding

    Two-way data binding (used mainly in template-driven forms) combines property and event binding in a single notation.
    Here's an example that uses two-way data binding with the ngModel directive.

    <input type="text" id="name" [(ngModel)]="name">


    In two-way binding, a data property value flows to the input box from the component as with property binding.
    The user's changes also flow back to the component, resetting the property to the latest value, as with event binding.
